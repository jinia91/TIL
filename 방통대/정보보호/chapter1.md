# 정보보호의 개념
- 정보를 여러 가지 위협으로부터 보호하기 위한 정책 및 기법
  - 정보의 상태 : 저장 , 전달
  - 위협의 종류 : 허락되지 않는 접근, 수정, 훼손, 유출 등

# 컴퓨터 보안의 개념
- 정보보호의 한 영역
- 컴퓨팅 환경이 관여된 모든 상황에 대한 정보보호

# 정보보호의 목표
- 정보보호의 핵심 목표
  - 기밀성(confidentiality)
  - 무결성(Integrity)
  - 가용성(Availability)
- CIA triad

## 기밀성 confidentiality

- 허락되지 않은자가 정보의 내용을 알 수 없도록 하는 것

- 접근 못하게 하기
- 접근하더라도 무의미한 내용만 보이게 하기 : 암호화


## 무결성 Integrity
- 허락되지 않은자가 정보의 내용을 임의로 수정할 수 없도록 하는 것
- 불가피하게 수정이 발견되었다면, 이를 즉시 확인할 수 있는 것

## 가용성 Availablity
- 허락된 자가 정보에 접근하고자 할 때, 방해받지 않도록 하는 것
- 필요할 때 언제든지 사용할 수 있어야 함

## 그외
### 부인 방지(non - repuidiation)
- 정보에 관여한 자가 이를 부인하는것을 방지
- 발신 부인방지 / 수신부인방지
  
### 인증(authentication)
- 어떤 실체가 정말 주장하는 실체가 맞는지 확인 / 신뢰할 수 있는 것
- 실체 : 정보 자체, 정보를 이용하는 사용자 등
  
### 접근 제어(access control) 
- 정보에 대한 허락된 접근만 허용하고, 그 외의 접근을 허용하지 않는 것
- 접근 권한이 있는자와 없는자 구분

# 정보화 환경과 역기능

## 컴퓨터의 역사와 위협
### 1980년대
- 모리스 웜 : 인터넷으로 연결된 수천대의 UNIX 컴퓨터를 감염
- 이를 계기로 침해대응센터인 CERT 가 만들어짐

### 1990년대
- 시티뱅크 시스템에 침입하여 자금 탈취
- 여러 회사 시스템에 침입하여 각종 정보 탈취
- 정부 시스템에 침입하여 걸프전 정보 탈취

### 2000년대
- 야후, CNN, 아마존 등 접속자가 많은 몇 개의 사이트에 분산 서비스 거부(DDos) 공격 발생
- 네트워크 상으 ㅣ취약한 서버를 찾아 미리 감염시킨 후, 이 서버들이 정해진 시간에 목표 사이트에 수많은 패킷을 전송하도록 함

### 2010년대 중반 랜섬웨어 유행
- 컴퓨터에 저장되어 있는 문서나 그림 파일 등을 암호화하여 사용자가 사용할 수 없게 만듬
- 암호를 풀기위해서는 비트코인등의 송금을 유도
