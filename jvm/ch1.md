# 자바 기술 시스템 소개

## 자바의 강점
- 엄격한 구조를 갖춘 객체 지향 프로그래밍 언어
- 한번 작성하면 어디서든 실행된다
- 안전한 메모리 관리 시스템
- 런타임 핫코드 감지, 컴파일(JIT)
- 풍부한 서드파티 오픈소스

## jvm 역사
### 1.0 클래식 vm
- 인터프리터 모드 / 컴파일모드 둘중 하나 선택만 가능, 컴파일 모드는 플러그인형태
- 컴파일 모드 실행시 jvm  실행 시스템 전체가 JIT 컴파일러에 넘어가는 구조로 모든 코드를 네이티브로 런타임에 컴파일해야했으므로 매우 느렸음


### 이그젝트 VM vs 핫스팟 VM
- 이그젝트 VM은 메모리의 특정 위치에 있는 데이터의 구체적인 자료형을 알고있던 VM
- 핫스팟 VM은 당시 SUN 사가 아니라 외부에서 만들었던 스트롱 토크용 VM을 인수해 개조
- 핫코드 감지 기술을 통해 지금에 우리가 알고있는 JIT  컴파일러 동작이 가능해짐


## jvm 미래

### graalvm
- 어디서든 더 빠르게 실행한다
- 각종 언어의 소스코드나 컴파일된 중간형식을 인터프리터를 통해 그랄vm이 이해할수 있는 중간 표현으로 변환하는식으로 동작
- 언어 프레임워크 트러플, llvm 비트코드 인터프리터 수롱
- 고성능 폴리글랏 가상 머신
- 기존 언어들의 런타임보다 더 빠른경우도 꽤 많다고 함

### 차세대 JIT 컴파일러

#### 기존 구성
- C1 : 컴파일 속도가 빠른 대신 최적화를 적게하는 클라이언트 컴파일러
- C2 : 컴파일 속도는 느리지만 더 많은 최적화를 적용한느 서버 컴파일러
- 인터프리터

#### 차세대 
- 그랄 컴파일러 : C2 대체 목적, 그랄vm으로 이식, 이후 jdk로 재이식

### 네이티브화 AOT
- 애플리케이션 실행전 네이티브 코드로 사전 컴파일해두는 기술

 ### 발할라 프로젝트
 - 값타입과 원시타이을 일반화한 제네릭 타입 제공, 불변 타입과 비참조 타입을 명시적으로 선언가능

### 파나마 프로젝트
- 네이티브 코드 호출시 더 편하게
